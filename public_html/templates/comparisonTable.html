Fact / Feature Objective-C Swift
Language Design Built on C/Superset of C. Lots of ugly syntax with "@" to denote objects or functions. Centralized around Apple platforms. Modern language with many features inspired from other popular languages. Designed to be platform agnostic, which makes it possible to run on non-Apple platforms.
Code Maintainability Separation of code using header files, similar to C. No namespaces. Drops separation based on header files. Better segregation of modules using namespace concept, with class keyword.
Safety Inappropriate null pointer handling. Using null pointers results in No-Op. Can use Optional Type to trigger a Runtime Error when accessing null pointers.
Memory Management Automatic Memory Management (using ARC) is not available across all frameworks. Example: programmer has to manually free all allocations when using the Graphics API. ARC support is integrated and available across all frameworks. This also enhances performance without having to run the Garbage Collector.
Code Structure for Class Declarations Requires separate interface and implementation declaration for each class. Results in code bloating and maintenance problems. Doesn't require separate interface from implementation; makes code simpler and easiser to read.
Data Structures and Return Types No support for Dictionaries or multiple returns. Enumeration similar to C, elements are integers. Dictionary, Tuple (and return Tuple) support built in. Adds improved version of "enum" that can handle strings, chars, ints, floats.
Programming Paradigms Modelled on Imperative Paradigm Lots of new features added to support functional programming (such as first-class functions, closures, pure functions etc).
Automatic Type Inference Every variable should have type explicitly declared (ex: NSString str = @"string") Supports type inferences. Example: var myStr = "hello"
Control Flow constructs Control flow constructs are similar to C. Provides control flow constructs, which simplifies code and reduces errors. Example: break statement no longer needed (but can be explicitly added if necessary). Support for a new guard construct to create if then else statements. Switch-case statement improved to support more patterns, interval matches, tuples, and casts.
Developer Productivity Similar to C/C++; Every time code is changed, recompilation is required to execute or test. Built in Playground feature to see results of code changes in real time.
Variable Names Need to be in ASCII, similar to C. Files can be written in Unicode with Unicode characters as name variables. Improves internationalization.
