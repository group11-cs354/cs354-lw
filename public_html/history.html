History of Swift

Early Development

Swift started out as a one man effort, but quickly grew into a multi-team collaboration. The language became an important aspect of the Developer Tools team at Apple. Chris Lattner is responsible for the early work on Swift. He has an impressive background; Chris is the original author of the LLVM Compiler Infrastructure and has his Ph.D. in Computer Science. He is also an integral member at Apple and has taken on various roles throughout his continued time at the company.

Chris Lattner, currently the Director of the Developer Tools department at Apple, started work on Swift in July of 2010.
During the next year, he worked on the basic language structure. Three years later, in July 2013, Swift became a priority focus for the Apple Developer Tools group. The language quickly became a large-effort collaboration, with involvement from various expert teams, including:
  • Language Team
  • Documentation Team
  • Compiler optimization Team
  • Feedback and Testing Team

Public Unveil

On June 2nd, 2014 at Apple's Worldwide Developer's Conference (WWDC), Swift was shown off to the world. Apple showed off a ton of exciting features and held multiple developer sessions to let developers at the show get a first glimpse of what to expect from the language. Apple also touted the Swift Playground, where you can edit code and see the results in realtime. They also announced that all of Swift's features would be built right into Xcode, the native IDE on OS X. Furthermore, they highlighted the readability and usability of the language for beginner or advanced programmers alike.

Open-Sourced

On June 8th, 2015, Apple announced Swift 2.0 at their WWDC. The main additions announced were:

  • Increased Performance
  • New API for Error Handling
  • First-Class Support for Availability Checking (ability to see if the API you're using is too new for target OS)
  • Protocol Extensions (global functions are replaced with methods to common types)
  • Syntax Improvements
In addition to these changes, Apple announced that Swift was going open source. The initial rollout included support for OS X and iOS.

Today: Swift 2.2

Swift 2.2 was released on March 21st, 2016. This was the first update to the language since it went open source. The update included contributions from 212 non-Apple developers that spanned across the language. The notable features from this update include:

  • Bug Fixes
  • Diagnostic Enhancements
  • Faster-running Code
  • Tuple Comparison Operators
  • Support for Swift on Linux (REPL and LLDB)

Future

Apple is expected to show off Swift 3 at their WWDC this year (2016) and discuss new changes to the language, as well as even greater support for Linux, including the Swift Core Libraries.
